# Ultra-Refine Workshops

When brainstorming stalls (2+ rejected proposals), deploy one workshop at a time. Pick the one that targets the specific failure mode:

| Workshop | When to use | Prompt |
|----------|------------|--------|
| **Inversion** | Proposals feel directionless | "What would make this document actively harmful?" |
| **Concrete Example** | Too abstract | "Walk me through one specific scenario where someone uses this." |
| **Five-Year-Old** | Overcomplicated | "Explain what this document is for to a five-year-old." |
| **Before/After** | Unclear value | "What does the workflow look like before and after this doc exists?" |
| **Worst Version** | Perfectionism blocking | "What's the laziest version that would still be useful?" |
| **Audience Test** | No clear reader | "Who reads this, and what will they do differently after?" |
| **Kill Test** | Might not be needed | "Should this exist at all? What happens if we delete it?" |

Rules:
- One workshop at a time — don't stack
- Wait for response before choosing next
- Kill Test concludes doc shouldn't exist → hand off to `/doc:maintain`
- Exit as soon as you have enough clarity for an accepted proposal
